# 🎮 The Gaming Room – Software Design Document

## 📊 Overview
This project is a software design document created for The Gaming Room, a client requesting the development of a web-based, multi-platform game called *Draw It or Lose It*. The game is a digital adaptation of a drawing and guessing game where teams compete to identify drawings in real time.

The application is designed to support multiple teams, multiple players per team, and enforce unique naming rules for games and teams. It also ensures that only one game instance exists in memory at a time to maintain system integrity.

---

## 🧩 Client & Requirements
The client, The Gaming Room, wanted a scalable web-based version of their existing game that could run across multiple platforms. Key requirements included:

- Support for multiple teams and players
- Unique names for games and teams
- Single active game instance in memory
- Cross-platform accessibility via web application architecture
- Scalable and maintainable system design

## 🎯 Design Approach
The system was designed using a structured and iterative approach. Requirements were first analyzed to identify core functionality and constraints, followed by the development of system architecture and class design. Emphasis was placed on scalability, modularity, and clarity of system structure to support long-term maintainability.
